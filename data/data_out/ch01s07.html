Современные подходы к разработке на Python

Глубокое понимание выбранного языка программирования - самая важная вещь, которую нужно использовать как эксперту. Это всегда будет актуально для любой техники. Тем не менее, действительно сложно разработать хорошее программное обеспечение, не зная общих инструментов и практик в рамках данного языкового сообщества. В Python нет единственной функции, которую нельзя было бы найти на каком-либо другом языке. Таким образом, при прямом сравнении синтаксиса, выразительности или производительности всегда будет решение, которое лучше в одном или нескольких областях. Но область, в которой Python действительно выделяется из толпы, - это целая экосистема, построенная вокруг языка. Его сообщество в течение многих лет совершенствовало стандартные методы и библиотеки, которые помогают создавать более надежное программное обеспечение в более короткие сроки.

Самая очевидная и важная часть упомянутой экосистемы - это огромная коллекция бесплатных пакетов с открытым исходным кодом, которые решают множество проблем. Создание нового программного обеспечения - всегда дорогостоящий и трудоемкий процесс. Возможность повторно использовать существующий код вместо того, чтобы изобретать колесо, значительно сокращает время и затраты на разработку. Для некоторых компаний это единственная причина, по которой их проекты экономически целесообразны.

По этой причине разработчики Python приложили много усилий для создания инструментов и стандартов для работы с пакетами с открытым исходным кодом, созданными другими. Начиная от виртуальных изолированных сред, улучшенных интерактивных оболочек и отладчиков до программ, которые помогают обнаруживать, искать и анализировать огромную коллекцию пакетов, доступных в PyPI ( индекс пакетов Python ).
