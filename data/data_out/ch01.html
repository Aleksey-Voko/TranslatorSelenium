Глава 1. Текущее состояние Python

Python хорош для разработчиков.

Независимо от того, какую операционную систему используете вы или ваши клиенты, она будет работать. Если вы не кодируете вещи для конкретной платформы или не используете библиотеку для конкретной платформы, вы можете работать в Linux и, например, развертывать в других системах. Однако это уже не редкость (точно так же работают Ruby, Java и многие другие языки). В сочетании с другими качествами, которые мы обнаружим в этой книге, Python становится разумным выбором в качестве основного языка разработки компании.

Эта книга ориентирована на последнюю версию Python, 3.5, и все примеры кода написаны на этой версии языка, если явно не указана другая версия. Поскольку этот выпуск еще не получил широкого распространения, эта глава содержит некоторое описание текущего статуса-кво Python 3 для ознакомления читателей с ним, а также некоторую вводную информацию о современных подходах к разработке на Python. В этой главе рассматриваются следующие темы:

  • Как поддерживать совместимость между Python 2 и Python 3
  • Как подойти к проблеме изоляции среды как на уровне приложения, так и на уровне операционной системы для целей разработки
  • Как улучшить подсказку Python
  • Как установить пакеты с помощью pip

Книга всегда начинается с закусок. Итак, если вы уже знакомы с Python (особенно с последней веткой 3.x) и знаете, как правильно изолировать среды для целей разработки, вы можете пропустить первые два раздела этой главы и просто быстро прочитать остальные разделы. Они описывают некоторые инструменты и ресурсы, которые не являются необходимыми, но могут значительно повысить производительность в Python. Обязательно прочтите раздел об изоляции среды на уровне приложений и pip, поскольку их установка является обязательной для остальной части книги.

Где мы сейчас и куда идем?

История Python начинается где-то в конце 1980-х годов, но дата его выпуска 1.0 была в 1994 году, так что это не очень молодой язык. Здесь может быть упомянута целая шкала основных выпусков Python, но действительно важна одна дата: 3 декабря 2008 г. - дата выпуска Python 3.0.

На момент написания прошло семь лет с момента первого выпуска Python 3. Также прошло четыре года с момента создания PEP 404 - официального документа, который «не выпустил» Python 2.8 и официально закрыл ветку 2.x. Хотя прошло много времени, в сообществе Python существует определенная дихотомия: хотя язык развивается очень быстро, есть большая группа его пользователей, которые не хотят двигаться вперед.
